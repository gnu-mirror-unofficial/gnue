Chapter 5. Location Module

5.1. Requirements

This chapter is $Revision: 1.1 $ $Date: 2003/01/05 00:00:29 $.

5.1.1. Business Requirements

  1. Maintain address information.

  2. Maintain a list of valid ISO country codes.

  3. Maintain a list of valid localized state (United States) or region codes.

5.1.2. Future Requirements

The following items will be implemented in future version of this module. They may be implemented in the base module or be added via an industry specific add on.

  • None

5.1.3. Business Objects

The following Business Objects are defined and maintained by this module.

  • location::address - TYPE only.

  • location::region - Define available regions.

  • location::country - Define available countries.

5.1.4. Forms

  • Add, change, delete, inquire regions.

  • Add, change, delete, inquire countries.

5.1.5. Reports

  • None

5.1.6. Business Rules

  • none

module location
{
  class post_code
  {
    char   zip_code<5>;
    char   city<15>;
    char   state<2>;
    float  latitude;
    float  longitude;
    
    UNIQUE INDEX(zip_code);
  };

  class region
  {
    char  country<8>;
    char  code<8>;   # state/region abbrev
    char  descr<35>;  # long state/region name
  };

  class country
  {
    char   phone_code<4>;    # for validating phone country code
    char   code<8>;          # for validating address country abbreviation
    char   description<35>;  # the long country name
    location::region []regions;        # regions or states within country
    char   city_format<35>;  # formatting template for the city/postal code
                             # address line
    
    UNIQUE INDEX(code);
  };

  # -------------------------------------------------------------------------
  # address - definition of physical addresses.
  # -------------------------------------------------------------------------
  type address
  {
    char    descr<35>;              # of this address (work, home, business etc.)
    char    street1<35>;
    char    street2<35>;
    char    street3<35>;
    char    city<35>;
    location::region  *region;      # state in the US
    location::country *country;
    char    postal<17>;
    
    # get the place as city, state, zip per local requirements
    # text get_place();
  };

};
